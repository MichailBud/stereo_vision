# Stereo Vision

**Курсовой проект** по предмету *Методы локализации, позиционирования и навигации*

## Тема проекта

**Алгоритм оценки положения мобильного робота по данным системы стереозрения**

### Цель
Локализация робота на основе триангуляции точек пространства с двух камер.

### Исходные данные
- Входные данные: стереопара изображений

### Задание
- Реализовать алгоритм поиска соответствий на эпиполярных линиях
- Построение карты диспаратности
- Самостоятельно вычислить 3D-координаты характерных точек сцены
- На их основе оценить перемещение камеры

### Итоговый результат
Оценка 6-DOF положения робота (позиция и ориентация).

### Технический стек
- ROS2 Humble
- Gazebo Classic
- Ubuntu 22.04

## Реализованный функционал

### ✅ Обработка стереоизображений
- Публикация выпрямленных изображений с левой и правой камеры
- Вычисление карты диспаратности алгоритмом **StereoSGBM**
- Встроенная speckle-фильтрация SGBM
- **Проверка синхронности кадров** — кадры с разницей > 50мс отбрасываются

### ⚠️ Фильтрация карты диспаратности (НЕ ИСПОЛЬЗУЕТСЯ)

Медианный фильтр и морфологическое закрытие **отключены** для повышения производительности:

```cpp
// Медианный фильтр для удаления импульсных выбросов
// cv::medianBlur(disparity, disparity, 5);

// Эрозия + дилатация (закрытие) для удаления шума и заполнения дыр
// applyMorphologicalClosing(disparity, morph_kernel_size);
```

### ✅ Настроенные параметры SGBM
| Параметр | Значение | Описание |
|----------|----------|----------|
| `sgbm_min_disparity` | 0 | Минимальная диспаратность |
| `sgbm_num_disparities` | 64 | Диапазон поиска (кратно 16) |
| `sgbm_block_size` | 7 | Размер окна сопоставления |
| `sgbm_p1` | 20 | Штраф за изменение на ±1 |
| `sgbm_p2` | 70 | Штраф за изменение > ±1 |
| `sgbm_disp12_max_diff` | 5 | Left-right проверка |
| `sgbm_uniqueness_ratio` | 15 | Фильтр уникальности |
| `sgbm_speckle_window_size` | 75 | Размер speckle-окна |
| `sgbm_speckle_range` | 4 | Диапазон speckle |

### ✅ 3D реконструкция
- Триангуляция точек по карте диспаратности
- Публикация облака точек (`/point_cloud2`)
- Визуализация в RViz

> **Важно:** OpenCV StereoSGBM возвращает disparity в формате `CV_16S` с субпиксельной точностью 1/16 (значения умножены на 16). При вычислении 3D-координат disparity делится на 16.0 для получения корректного масштаба в метрах: `Z = (B * fx) / (disp / 16.0)`.

### ✅ Фильтрация облака точек (pointcloud_filter)

| Этап | Описание |
|------|----------|
| 1. Удаление NaN/нулей | Отбрасываются точки с NaN координатами или нулевыми значениями |
| 2. VoxelGrid | Downsampling (по умолчанию leaf=0.1) |
| 3. SOR | Statistical Outlier Removal (MeanK=50, Stddev=1.0) |
| 4. Transform | Преобразование из camera_frame в base_link |

**Параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `voxel_leaf` | 0.1 | Размер вокселя (м) |
| `sor_mean_k` | 50 | Количество соседей для SOR |
| `sor_stddev_mul_thresh` | 1.0 | Порог SOR |
| `cam_to_base_x` | 0.2 | Смещение камеры по X |
| `cam_to_base_y` | 0.0 | Смещение камеры по Y |
| `cam_to_base_z` | 0.055 | Смещение камеры по Z |

### ✅ ICP SLAM с EKF (icp_slam)

Пакет `icp_slam` объединяет одометрию колёс с ICP визуальной одометрией через расширенный фильтр Калмана (EKF).

**Особенности:**
- **Только позиция (x, y) от ICP** — углы (yaw/pitch/roll) берутся из одометрии
- **Multiscan режим** — накопление глобальной карты окружения
- Только **VoxelGrid** фильтрация (SOR отключён)
- **Проверка качества ICP** — коррекция EKF только при `fitnessScore < 0.5`
- **Разделение ICP и накопления карты** — облако добавляется в карту ПОСЛЕ коррекции EKF

**Алгоритм работы:**
1. **Шаг 1:** Предсказание EKF + ICP между последовательными облаками
2. **Шаг 2:** Коррекция EKF с использованием ICP (с поворотом на угол робота)
3. **Шаг 3:** Добавление облака в глобальную карту с исправленной позой

**Ключевые исправления:**
- Локальное смещение ICP преобразуется в глобальные координаты через поворот: `z = x_hat + R * [dx, dy]`
- Карта трансформируется в глобальную систему координат (map) перед накоплением
- Используется скорректированная поза робота (после EKF) для построения карты

**Параметры:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `MaximumIterations` | 50 | Макс. итераций ICP |
| `MaxCorrespondenceDistance` | 0.1 | Макс. дистанция соответствия (м) |
| `Leaf` | 0.1 | Размер вокселя для downsampling (м) |
| `motion_noise` | [0.01, 0.01, 0.001] | Шум модели движения |
| `measurement_noise` | [0.05, 0.05, 0.01] | Шум измерений ICP |

**Топики:**
| Топик | Тип | Описание |
|-------|-----|----------|
| `/filtered_cloud` | `sensor_msgs/PointCloud2` | Входное облако (от pointcloud_filter) |
| `/odom` | `nav_msgs/Odometry` | Одометрия колёс |

| `/ekf_pose` | `geometry_msgs/PoseStamped` | Поза робота от EKF |

## История разработки и ключевые исправления

### Проблемы и решения

1. **Проблема: карта "двигалась" вместе с роботом**
   - Причина: подавалось локальное смещение от ICP как глобальная координата
   - Решение: преобразование через поворот `z = x_hat + R * [dx, dy]`

2. **Проблема: карта строилась в base_link, а не в map**
   - Причина: облака не трансформировались в глобальные координаты
   - Решение: добавлена трансформация через `robot_pose`

3. **Проблема: размазывание карты**
   - Причина: облака добавлялись в карту ДО коррекции EKF (с предсказанной позой)
   - Решение: разделены `computeICPMatch` и `addCloudToMap`, карта строится ПОСЛЕ коррекции

4. **Проблема: некорректная коррекция EKF при плохом ICP**
   - Причина: EKF корректировался даже при несходящемся ICP
   - Решение: добавлена проверка `fitnessScore < 0.5`

### Архитектура ScanMatcher

```
ScanMatcher
├── computeICPMatch()  - только ICP, без накопления карты
├── addCloudToMap()     - добавление облака в карту с заданной позой
├── getLastICPTransform() - получить последнюю трансформацию
├── getFitnessScore()    - получить качество ICP
└── getMergedPointCloud() - получить глобальную карту
```

### Алгоритм в timer_callback

```
1. predict(u_t)                    # Предсказание EKF
2. computeICPMatch()               # ICP между облаками
3. z_t = x_hat + R * icp_offset    # Преобразование в глобальные координаты
4. if (fitness < 0.5) correct(z_t) # Коррекция EKF (только при хорошем ICP)
5. addCloudToMap(robot_pose_corrected)  # Добавление в карту с исправленной позой
6. publishTF()                     # Публикация map -> odom
```
